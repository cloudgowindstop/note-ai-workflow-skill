# PDF 学习笔记助手

## ⚠️ 关键规则

1. **必须使用 MCP 工具处理 PDF**：提取内容用 `pdf_extract`，截图用 `pdf_screenshot`。禁止使用 `pdftotext`、`pdfinfo` 等系统命令。
2. **智能截图**：页面包含图表、公式、表格、代码块，或 pdfplumber 提取文本不完整时 → 必须截图。纯文字页面且提取完整 → 可跳过截图。
3. **必须写入文件**：所有内容必须写入 `./notes/` 目录，不能只在对话中展示。

---

## 两种工作模式

开始工作前，先判断 `./notes/{pdf文件名}/study_notes.md` 是否已存在：

- **不存在** → 模式 A：首次分析
- **已存在** → 模式 B：深度追问

---

## 模式 A：首次分析

用于初次阅读一篇 PDF，产出完整的结构化笔记。

### A1. 确认 PDF 路径
- 从用户消息中提取 PDF 文件名
- 转为绝对路径

### A2. 提取内容
- 调用 `pdf_extract` 获取全部页面文本

### A3. 定位关键页面
- 基于用户问题语义匹配，选出最相关的 3-5 页

### A4. 智能截图
- 检查每页内容：有图表/公式/表格/代码 → 调用 `pdf_screenshot`
- 纯文字且提取完整 → 可跳过
- `output_dir` 设为 `./notes/{pdf文件名}/screenshots/`

### A5. 生成笔记
对每个关键页面生成五个部分：
1. **翻译**：学术英文 → 流畅中文，专业术语保留英文原文
2. **讲解**：通俗解释；公式拆解每个符号（LaTeX）；图表描述信息
3. **重点提炼**：3-5 个核心要点
4. **关联问题**：2-3 个可延伸思考的问题
5. **补充知识**：相关概念或推荐阅读

### A6. 写入 study_notes.md
```markdown
# {PDF文件名} 学习笔记

> 日期：{当前日期}

---

## 第 N 页 — {章节标题}

<!-- 如有截图 -->
![page_N](screenshots/page_N.png)

### 翻译
{中文翻译，专业术语保留英文原文}

### 讲解
{通俗解释。公式 LaTeX。图表描述。}

### 重点提炼
- {要点 1}
- {要点 2}
- {要点 3}

### 关联问题
1. {延伸思考 1}
2. {延伸思考 2}

### 补充知识
- {相关概念或推荐阅读}

---
```

## 模式 B：深度追问

用户对已分析过的论文提出进一步问题。产出按知识模块组织的 Q&A 笔记。

### B1. 识别追问类型

| 类型 | 特征 | 归属模块 |
|------|------|---------|
| **背景知识** | "xxx 是什么"、"如何训练"、"为什么" | 背景知识补全 |
| **方法深入** | 针对论文具体方法的追问 | 方法细节深挖 |
| **延伸对比** | 与其他方法的比较、"为什么不" | 延伸思考 |
| **工程实践** | 训练细节、实现方式、数据集 | 工程实现 |
| **其他** | 不属于以上 | 补充讨论 |

### B2. 读取已有笔记
- 用 `Read` 读取 `./notes/{pdf文件名}/study_notes.md` 获取上下文
- 用 `Read` 读取 `./notes/{pdf文件名}/deep_dive.md`（如存在）获取已有追问

### B3. 必要时回查 PDF
- 如果问题涉及原文细节，调用 `pdf_extract` 重新定位相关页面
- 如果涉及图表/公式，调用 `pdf_screenshot` 补截图

### B4. 生成回答
对每个追问：
1. **直接回答**：先给核心结论（2-3 句）
2. **详细讲解**：层层展开，通俗 + 严谨并重
3. **联系论文**：将答案与论文中的具体方法/实验关联

### B5. 写入 deep_dive.md
追加到 `./notes/{pdf文件名}/deep_dive.md`，按模块归类：

```markdown
# 深度追问 — {PDF文件名}

---

## {模块名}

### Q: {用户的追问原文}

**核心回答**
{2-3 句直接结论}

**详细讲解**
{层层展开。公式用 LaTeX。}

**与论文的关联**
{这个知识点在论文第 X 页，对应 {具体方法/实验}。补充截图如需要。}

---
```

### B6. 追加策略
- 先读 `deep_dive.md`，判断新问题属于已有模块还是新模块
- 已有模块 → 追加在该模块末尾
- 新模块 → 创建新模块标题，追加在文件末尾
- 每次追问用 `---` 分隔

