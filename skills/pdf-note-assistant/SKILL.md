# PDF 学习笔记助手

## 触发条件

当用户提问包含以下关键词或场景时，本 Skill 自动激活：
- 提到 **PDF、论文、课件、教材、文献**
- 要求对英文内容做 **翻译**
- 需要 **整理笔记** 或 **截图**
- 要求 **解释学术概念**、公式或图表

## 工作流程（SOP）

### 步骤 1：理解用户意图
- 解析用户问题，提取核心概念和关键词
- 确认目标 PDF 文件路径（支持相对路径，如 `./papers/lecture.pdf`）
- 将相对路径转为绝对路径后再调用 MCP 工具

### 步骤 2：提取 PDF 内容
- 调用 `pdf_extract` 获取全部页面文本
- 注意标记为「此页无文字内容」的页面（扫描版 PDF），后续通过截图分析

### 步骤 3：语义定位相关页面
- 基于用户问题在全文上进行语义分析
- 找到最相关的页面（默认取 top 3-5 页）
- 判断依据：概念匹配度、关键词密度、内容相关性

### 步骤 4：截取页面截图
- 对每个定位到的页面，调用 `pdf_screenshot`
- `output_dir` 设为 `./notes/{pdf文件名}/screenshots/`
- 每页一张截图：`page_{N}.png`

### 步骤 5：生成笔记
对每个相关页面，生成以下五个部分：

1. **翻译**：将学术英文翻译为流畅中文，保留专业术语英文原文（如 L1 regularization、backpropagation）
2. **讲解**：通俗解释概念；公式解释每个符号含义；图表描述传达的信息
3. **重点提炼**：3-5 个核心要点
4. **关联问题**：2-3 个延伸思考题
5. **补充知识**：相关概念或推荐阅读

### 步骤 6：保存结构化笔记
- 写入 `./notes/{pdf文件名}/study_notes.md`
- 使用相对路径引用截图（`screenshots/page_N.png`）
- 目录不存在时自动创建

## 输出模板

```markdown
# {PDF文件名} 学习笔记

> 问题：{用户提问摘要}
> 日期：{当前日期}

---

## 第 N 页 — {章节标题}

### 原文截图

![page_N](screenshots/page_N.png)

### 翻译

{中文翻译，专业术语保留英文原文并用括号标注，如「注意力机制（Attention Mechanism）」}

### 讲解

{用通俗语言解释概念。如果是公式，拆解每个符号含义，使用 LaTeX 语法。如果是图表，描述其传达的信息。}

### 重点提炼

- {要点 1}
- {要点 2}
- {要点 3}

### 关联问题

1. {延伸思考 1}
2. {延伸思考 2}

### 补充知识

- {相关概念或推荐阅读}

---
```

## 质量检查清单

- [ ] 截图包含问题相关的完整上下文
- [ ] 翻译准确，专业术语保留英文
- [ ] 讲解让非专业人士也能理解
- [ ] 公式使用 LaTeX 语法（`$...$` 行内，`$$...$$` 块级）
- [ ] 截图路径正确（相对路径 `screenshots/page_N.png`）
- [ ] 笔记保存到 `./notes/{pdf文件名}/study_notes.md`
- [ ] 每页包含完整的五个部分

## 注意事项

- PDF 文件的路径请使用绝对路径传给 MCP 工具
- 扫描版 PDF（无文字层）可直接从截图分析内容
- 公式、代码块等特殊内容注意格式正确
- 一个 PDF 的多次提问，追加到同一 `study_notes.md`（用 `---` 分隔）
