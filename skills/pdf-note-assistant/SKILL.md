# PDF 学习笔记助手

## ⚠️ 关键规则（必须遵守）

1. **必须使用 MCP 工具处理 PDF**：不要使用 `pdftotext`、`pdfinfo` 等系统命令。提取 PDF 内容只能用 `pdf_extract`，截取页面只能用 `pdf_screenshot`。
2. **每页都要截图**：定位到的每个相关页面，必须调用 `pdf_screenshot` 截图，截图是笔记的必要组成部分。
3. **必须写入文件**：笔记内容必须用 `Write` 工具保存到 `./notes/{pdf文件名}/study_notes.md`，不能只在对话中展示。

## 触发条件

当用户提问涉及以下任意场景时，本 Skill **自动激活**：
- 提到 PDF 文件路径或文件名（如 `xxx.pdf`）
- 提到论文、课件、文献、教材、paper
- 要求分析、翻译、解释 PDF 内容
- 要求整理笔记、截图

## 工作流程（SOP）

### 步骤 1：确认 PDF 路径
- 从用户消息中提取 PDF 文件路径
- 将相对路径转为**绝对路径**（使用 `pwd` 确认当前目录，拼接完整路径）

### 步骤 2：提取内容
- 调用 `pdf_extract` 工具，传入 PDF 的绝对路径
- 获取全部页面文本

### 步骤 3：定位关键页面
- 基于用户问题，在全文上进行语义匹配
- 选出最相关的 3-5 页

### 步骤 4：截图
- 对每个关键页面，调用 `pdf_screenshot`
- `output_dir` 参数设为 `./notes/{pdf文件名}/screenshots/`
- 截图文件自动命名为 `page_{N}.png`

### 步骤 5：生成笔记
对每个关键页面生成以下五个部分：
1. **翻译**：学术英文 → 流畅中文，专业术语保留英文原文
2. **讲解**：通俗解释；公式拆解每个符号（LaTeX 格式）；图表描述传达的信息
3. **重点提炼**：3-5 个核心要点
4. **关联问题**：2-3 个延伸思考题
5. **补充知识**：相关概念或推荐阅读

### 步骤 6：写入文件
- 使用 `Write` 工具保存到 `./notes/{pdf文件名}/study_notes.md`
- 截图引用使用相对路径：`screenshots/page_N.png`
- 目录不存在时用 `mkdir -p` 创建

### 原文截图

### 关联问题

## 质量检查

完成笔记后自查：
- [ ] 每个页面都调用了 `pdf_screenshot` 并包含截图
- [ ] 笔记已用 `Write` 写入 `./notes/` 目录
- [ ] 截图使用相对路径引用
- [ ] 公式使用 LaTeX 语法
- [ ] 专业术语保留英文原文
