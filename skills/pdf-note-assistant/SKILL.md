# PDF 课程学习助手

一个 Skill，四种模式，共享 `./notes/` 知识库。

---

## 关键规则

1. **必须使用 MCP 工具处理 PDF**：提取内容用 `pdf_extract`，截图用 `pdf_screenshot`。禁止使用 `pdftotext`、`pdfinfo` 等系统命令。
2. **智能截图**：页面包含图表、公式、表格、代码块，或 pdfplumber 提取不完整 → 必须截图。纯文字且提取完整 → 可跳过。
3. **必须写入文件**：所有内容写入 `./notes/` 目录，不能只在对话中展示。
4. **多 PDF 感知**：每次开始前先扫描 `./notes/README.md`，了解本课程已有的全部 PDF 笔记和文件结构。
5. **自动维护索引**：每次写入新笔记后，更新 `./notes/README.md`。
6. **主干与追问分离**：首次分析写入 `notes.md`（论文框架），深度追问写入 `deep_dive.md`（探索记录）。两个文件定位不同，不混合。

---

## 模式判断

开始工作前，按以下顺序判断进入哪个模式：

```
1. 用户提到作业/习题/题目？                         → 模式 D（作业分析）
2. 用户提到 2 个及以上 PDF 或明确要求对比？          → 模式 C（交叉对比）
3. ./notes/{pdf名称}/notes.md 已存在？              → 模式 B（深度追问）
4. 以上都不满足？                                    → 模式 A（首次分析）
```

---

## 模式 A：首次分析

初次阅读一篇 PDF，产出结构化主干笔记。

### A1. 确认路径 & 初始化
- 提取 PDF 文件名，转为绝对路径
- 创建 `./notes/{pdf名称}/` 和 `./notes/{pdf名称}/screenshots/` 目录
- 扫描 `./notes/README.md` 了解课程索引状态

### A2. 提取内容
- 调用 `pdf_extract` 获取全部页面文本

### A3. 定位关键页面
- 基于用户问题语义匹配，选出最相关的 3-5 页

### A4. 智能截图
- 检查每页：有图表/公式/表格/代码 → 调用 `pdf_screenshot`
- 纯文字且提取完整 → 可跳过
- `output_dir` = `./notes/{pdf名称}/screenshots/`
- **重要**：只在确实截了图的页面才加 `![page_N]` 行，没截图就不要写空白图片占位

### A5. 生成笔记
对每个关键页面生成四个部分：
1. **翻译**：学术英文 → 流畅中文，专业术语保留英文原文
2. **讲解**：通俗解释；公式拆解符号（LaTeX）；图表描述信息
3. **重点提炼**：3-5 个核心要点
4. **补充知识**：相关概念或推荐阅读

### A6. 写入 notes.md
使用 `Write` 保存到 `./notes/{pdf名称}/notes.md`：

```markdown
# {PDF文件名} 学习笔记

> 日期：{当前日期}

---

## 第 N 页 — {章节标题}

<!-- 仅当本页调用了 pdf_screenshot 并生成了截图时，才包含下面这行 -->
![page_N](screenshots/page_N.png)

### 翻译
{中文翻译，专业术语保留英文原文}

### 讲解
{通俗解释。公式 LaTeX。图表描述。}

### 重点提炼
- {要点 1}
- {要点 2}
- {要点 3}

### 补充知识
- {相关概念或推荐阅读}

---
```

### A7. 更新索引
更新 `./notes/README.md`（见末尾「索引维护」章节）。

## 模式 B：深度追问

对已有笔记的 PDF 提出进一步问题。写入**独立的 `deep_dive.md` 文件**，与主干 `notes.md` 分离。

设计意图：
- `notes.md` = 论文"地图"，适合快速回顾框架
- `deep_dive.md` = 个人"探索记录"，适合深入研究和备考

### B1. 读取上下文
- `Read` 读取 `./notes/{pdf名称}/notes.md`（了解论文框架）
- `Read` 读取 `./notes/{pdf名称}/deep_dive.md`（如存在，了解已有追问）
- `Read` 读取 `./notes/README.md`
- 全部读完后再开始回答，不要分步读

### B2. 必要时回查 PDF
- 如果问题涉及原文细节，调用 `pdf_extract` 重新定位相关页面
- 如涉及图表/公式，调用 `pdf_screenshot` 补截图

### B3. 判断问题归属模块

| 类型 | 特征 | 模块名 |
|------|------|--------|
| 背景知识 | "xxx 是什么"、"如何训练"、"为什么" | 背景知识补全 |
| 方法深入 | 针对论文具体方法的追问 | 方法细节深挖 |
| 延伸对比 | 与其他方法比较、"为什么不" | 延伸思考 |
| 工程实践 | 训练细节、实现方式、数据集 | 工程实现 |
| 其他 | 不属于以上 | 补充讨论 |

### B4. 写入 deep_dive.md
将 Q&A 写入 `./notes/{pdf名称}/deep_dive.md`：

```markdown
# 深度追问 — {PDF文件名}

---

## 模块：{模块名}

### Q: {用户追问原文}

**核心回答**
{2-3 句直接结论}

**详细讲解**
{层层展开。公式 LaTeX。从 PDF 中引用具体段落。}

**与论文的关联**
{这个知识点对应论文第 X 页的 {具体方法/实验}。}

---
```

### B5. 追加策略
- 先读 `deep_dive.md`（如存在），判断新问题属于已有模块还是新模块
- 已有模块 → 追加在该模块末尾
- 新模块 → 创建新模块标题
- 每次追问以 `---` 分隔

### B6. 更新索引
`deep_dive.md` 首次创建时，更新 `./notes/README.md` 索引。

## 模式 C：交叉对比

对两个或更多 PDF 进行横向对比分析。

### C1. 识别涉及的 PDF
- 从用户问题中提取所有 PDF 名称
- 如果用户说"对比这篇和之前那篇"，从 `./notes/README.md` 中推断具体 PDF
- 确认每个 PDF 的 `notes.md` 存在，如不存在先执行模式 A

### C2. 并行获取信息
- 对每个涉及的 PDF 调用 `pdf_extract`
- 同时 `Read` 各自的 `notes.md` 和 `deep_dive.md`（如存在）获取已有分析

### C3. 生成对比
对每个对比维度生成表格 + 深入分析：

```markdown
| 维度 | {PDF A 名称} | {PDF B 名称} |
|------|-------------|-------------|
| 问题定义 | ... | ... |
| 核心方法 | ... | ... |
| 关键创新 | ... | ... |
| 实验设置 | ... | ... |
| 局限性 | ... | ... |
```

### C4. 写入 cross_topics.md
追加到 `./notes/cross_topics.md`：

```markdown
# 跨论文对比与综合讨论

---

## 对比：{A} vs {B}

> 日期：{当前日期}

### {用户的具体问题}

{对比表格 + 分析}

### 总结

{综合结论，两条论文的关系、各自的适用场景、互补性}

---
```

### C5. 更新索引
更新 `./notes/README.md`。

## 模式 D：作业分析

在全部课程 PDF 的上下文中回答作业题。

### D1. 理解题目
- 解析作业问题，拆解为多个子问题
- 从 `./notes/README.md` 获取课程全部 PDF 列表

### D2. 全课程搜索
- 调用 `pdf_extract` 搜索与题目相关的 PDF 页面
- `Read` 读取所有相关 `notes.md` 和 `deep_dive.md` 的已有分析
- 如有需要，调用 `pdf_screenshot` 截取证据

### D3. 综合回答
对每个子问题：
1. **核心结论**（2-3 句）
2. **详细推导**（引用具体 PDF 的原文和页码）
3. **关键截图**（如涉及公式/图表）

### D4. 写入 homework.md
追加到 `./notes/homework.md`：

```markdown
# 作业记录

---

## 题目：{作业问题摘要}

> 日期：{当前日期}
> 涉及材料：{相关 PDF 列表}

### 子问题 1：{拆解的题目}

**核心结论**
{2-3 句}

**详细推导**
{结合论文原文，引用页码和截图}

### 子问题 2：...
...
```

### D5. 更新索引
更新 `./notes/README.md`。

## 输出文件结构

```
./notes/
├── README.md              ← 课程笔记索引（自动维护）
├── {paper-a}/             ← 论文 A
│   ├── notes.md           ← 主干笔记：论文框架和讲解
│   ├── deep_dive.md       ← 深度追问：按模块组织的 Q&A
│   └── screenshots/
├── {paper-b}/             ← 论文 B
│   ├── notes.md
│   ├── deep_dive.md
│   └── screenshots/
├── {lecture-x}/           ← 课件/其他 PDF
│   ├── notes.md
│   ├── deep_dive.md
│   └── screenshots/
├── cross_topics.md        ← 跨 PDF 对比与综合讨论
└── homework.md            ← 作业分析
```

## 索引维护

每次写入笔记后，自动更新 `./notes/README.md`。格式：

```markdown
# {课程名} 学习笔记索引

> 最后更新：{当前日期时间}

## PDF 笔记

| 文件 | 主题 | 状态 |
|------|------|------|
| [论文A](./paper-a/notes.md) | {一句话主题} | 主干 + 追问([deep_dive](./paper-a/deep_dive.md)) |
| [论文B](./paper-b/notes.md) | {一句话主题} | 主干 |
| [课件X](./lecture-x/notes.md) | {一句话主题} | 主干 + 追问([deep_dive](./lecture-x/deep_dive.md)) |

## 交叉对比

- [跨论文讨论](./cross_topics.md) — {最近对比主题}

## 作业

- [作业记录](./homework.md) — {最近作业题目}
```

课程名从当前工作目录名推断。

## 质量检查

每次工作完成后自查：
- [ ] 使用了 MCP 工具而非系统命令（`pdftotext` 等）
- [ ] 有图表/公式的页面已截图
- [ ] 内容已写入正确的文件（A→notes.md，B→deep_dive.md，C→cross_topics.md，D→homework.md）
- [ ] 主干 notes.md 与追问 deep_dive.md 已分离，没有混合
- [ ] 公式使用 LaTeX 语法
- [ ] 专业术语保留英文原文
- [ ] `./notes/README.md` 索引已更新
- [ ] 模式 B 追问已按模块归类，模块名合理
